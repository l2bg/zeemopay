01

Three players.
One infrastructure.

⚡
Player 1 — Your customer
Tool Developer
You. You built an API, an LLM wrapper, a data tool, a specialized service. You want to charge per call and get paid instantly — without becoming a payment infrastructure engineer.
"I just want to monetize my tool. I don't want to touch wallets, blockchain, or payment rails."
🤖
Player 2 — The buyer
AI Agent
An autonomous AI agent that needs to call paid tools to do its job. It has a crypto wallet with USDC. It cannot use credit cards, cannot do OAuth, cannot fill out a checkout form. It pays programmatically or not at all.
"I need to pay for API calls with no humans in the loop. Stripe doesn't work for me."
Z
Player 3 — The infrastructure
Zeemo
The payment layer between agents and tools. Zeemo intercepts the request, verifies USDC on-chain, triggers your callback, splits the money instantly, and writes a tamper-evident receipt. Neither player builds this. Zeemo is this.
You get paid 95% the moment the agent calls. No invoices. No chargebacks. No payment code.

Step 1 — Register
# Register your tool — one call, you're live
curl -X POST \
  https://zeemopay-production.up.railway.app/register \
  -H "Content-Type: application/json" \
  -d '{
    "wallet_address": "0xYOUR_WALLET",
    "tool_name":      "your-tool-name",
    "price_per_call": "0.10",
    "callback_url":   "https://your-server.com/tool",
    "timeout_seconds": 10
  }'

# Response
{
  "status":        "registered",
  "pay_endpoint":  "/pay/your-tool-name",
  "price_per_call": "0.10"
}
